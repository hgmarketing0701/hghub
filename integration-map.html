🧩 The architecture in one line

There is one shared Job Record, keyed by Job ID, living in your Sheet + Apps Script backbone.
Every system reads the colours it needs and writes only its own colour back to that record.
No system talks to another directly — they all talk to the record. That is the whole integration.

1 · The two keys that hold it together

Everything joins on these. Get them right first — the rest is just columns.

CLIENT-0001

Client ID

One per client. Carries BLUE (name, tel, contact, comms). A client can have many jobs.

Minted at · CRM (Stage 2)
JOB-0001-A

Job ID

One per job / lot. Links back to a Client ID. Every colour from GREEN onward attaches here.

Minted at · Smart Quotation (Stage 3)

4 · What to wire first

The platform only "connects" once the keys and the shared record exist. Order matters.

1
Lock the Job Record schemado first
One Sheet (or one Apps Script data layer) with a column block per colour, keyed by Client ID + Job ID. This is the contract every system signs.
2
Build the CRM (Stage 2)do first
It mints Client ID and writes Blue. Nothing downstream connects until Blue exists in one place. (See the CRM spec.)
3
Wire CRM → Smart Quotationnext
Quotation reads Blue by Client ID, mints Job ID, writes Green. Your first real "two systems talking" win.
4
Point each existing system at the Job Recordnext
Permit, Pre-Onsite, Factory, Onsite, JCR, Backend already exist — retrofit each to read by Job ID and write its one colour per the matrix.
5
WATI inbound + outboundwhen method decided
Webhook leads into CRM, send quotes / updates out. Slot in once you've chosen the WATI method.
