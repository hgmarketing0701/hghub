Build Order — wire in this sequence
Lock the Job Record schema — one column block per colour, keyed by Client ID + Job ID. The contract every system signs.
Build the CRM (Stage 2) — mints Client ID, writes Blue. Nothing connects until Blue lives in one place.
Wire CRM → Smart Quotation — reads Blue by Client ID, mints Job ID, writes Green. First real win.
Retrofit existing systems — Permit, Pre-Onsite, Factory, Onsite, JCR, Backend read by Job ID, write one colour each.
WATI inbound + outbound — webhook leads in, send quotes/updates out. Slot in once method decided.
Transport — how data actually moves
WATI → CRM: webhook → Apps Script doPost(e) → append row, mint Client ID
CRM → Quotation: reads Blue by Client ID, mints Job ID · same Sheet, no export
Quotation → Client: auto-send quote + stored measurement via MailApp / WATI
Any system → Job Record: read by Job ID, write only own colour columns
Permit → Building mgmt: auto-email permit + docs + MOS + HIRARC · replies → v2, v3…
Job Completion → Client/Wix/LinkedIn: email + save-mode social drafts for manual post
Backend → CIMS/Infotech: prepares summary, human submits (manual for now)
